## R

* What is the need for a `path_to_models` argument?
* Use `setReplaceMethod` instead of `cell_type<-` if you are creating an S4
method.
* We would recommend that you use an empty environment to load data rather
than the function environment to avoid conflicts and messages from
uninitialized objects (in `load_models`):

```r
data_env <- new.env(parent = emptyenv())
data("default_models", envir = data_env)
default_models <- data_env[["default_models"]]
```
* We wouldn't recommend creating methods for classes that are not your own
because it would pollute that class's method space, e.g.,
`setMethod("check_parent_child_coherence", c("obj" = "Seurat"), ...` and
for `SingleCellExperiment`. It may even be better to reconcile both methods
into one that would handle both class inputs.
* Avoid using `/` in file paths as this is not portable for all systems, use
`file.path` instead.
* Do not write to the user's directory without explicit permission (e.g.,
`models.file.path` and `path.to.models`)
* Use a temporary directory by default in `path.to.models`.
* Consider using `BiocFileCache` to manage these saved instances.
* It would be better to let users save their own `.Rda` files rather than
providing a function like `save_new_model` especially if it does not provide
much functionality outside of `save`.

> We are afraid that reconciling both methods into one would generate many much
more complicated bugs. Therefore, we feel that we arrive at a cleaner code
base if we use dedicated functions for these objects. As both a very
established already, it's unlikely that additional basic objects of scRNA-seq
data will be developed.

When you have two versions of similar code, you would have twice the bugs.
Putting them into one function will reduce the places you have to debug.
Again, these classes are not yours so you shouldn't be adding methods to
their namespace. The appropriate place for additional methods is in the
package where the classes are defined. Otherwise, this will cause confusion
among users. Please remove these methods and make a single plain function that
processes both class inputs and returns the `return_val` list.
