* Any reason why `ReadH5AD` is changing the class of the original object?
In the example for `ReadH5AD`,  the original object is a `SummarizedExperiment`
but when it's read in, it becomes a `SingleCellExperiment`. Any user would
expect it to be the same class.
```r
WriteH5AD(miniACC[[1]], "miniacc.h5ad")
sce <- ReadH5AD("miniacc.h5ad")
```
* The data format should not change the class memberships, this also happens
with the `ReadH5MU` function on the `MultiAssayExperiment` where everything
becomes a `SingleCellExperiment`.
* It is recommended to keep the order of the assays the same as the original
`MultiAssayExperiment` after saving and reading.
* Avoid creating methods for classes that are not your own. Create plain
functions for writing classes (`WriteH5MU`) similar to `write_data_frame` and
`write_matrix`. This will avoid cluttering of the class's namespace.
* Preserve the metadata from the original object e.g., `metadata(miniACC)`
