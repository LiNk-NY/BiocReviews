separating functions by their return type so that they are "pure". It's not
recommended to have mixed return values / classes within a function
* Ideally you'd have an 'endomorphic' operation where the input class matches
the output class
* Avoid `glue` and its dependencies by using `sprintf()`
* Formatting code to fit within 80 column widths makes it easier to maintain
and read
* Use `message` instead of `print`
* Fix typo in `R/utils.R#83` which says `ExpressionSet` instead of
`SummarizedExperiment`
* Avoid the verbose output of `tensorflow::install_tensorflow()` (and the long
loading time) by removing it from the `.onLoad` and asking users to run it
separately when running `cellassign` by possibly using a wrapper function so
that the user can set `verbose = TRUE` / `FALSE`

before it is required and have a package option where something like
`verboseInstall = FALSE`
