# SEMplR #3962

* Consider renaming the package so that the lower case "l" is not confused
  with the numeral "1" or capital "I". Consider using all capital letters
  (SEMPLR) or changing the "l" to a different letter.
* A significant portion of the code is dedicated to plotting functions.
  Consider separating plotting functions into their own package to improve
  modularity.

## DESCRIPTION

* Looks good.

## NAMESPACE

* Consider separating score calculation and plotting functions into their own
  respective packages to improve modularity and ease of maintenance.

## vignettes

* Consider only using one vignette output type (e.g., only `BiocStyle`)

## R

* Uncomment the validity method for `SEMplScores` or remove it if not needed.
* Add the `@source` for the `SEMC` dataset.
* It is preferrable to use `function()` instead of the more cryptic `\( )`
  syntax for defining named functions, especially in package code
  (`R/enrichSEMs.R`).
* Where possible, use named indices rather than numeric indices for accessing
  elements of lists and data frames (e.g., `sem[["value"]]` rather than
  `sem[, 1]` in `R/loadSEMcollection.R`).
