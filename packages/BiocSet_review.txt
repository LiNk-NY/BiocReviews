## R

* Is the `GMTFile` class necessary? It may be simpler to use a GMT file path
string unless you have several methods for the GMTFile class.
* Provide a coercion method from `GeneSetCollection` or `getGmt` to `BiocSet`
(if not present already).
* How closely do you want the functions to match `dplyr`? If quite close,
`filter_element` would have to operate based on (potentially) three subclasses
and be called `filter`.
* `update_es_element` and friends seems a little circular, it may just be
better to use `es_element<-` from the get-go
* Do you want to export the replace methods? If so, consider using
`setReplaceMethod`
* Export the coercion method `setAs` as well.
* It feels like the `*_elementset` / `*_set` definitions should not get in the
way of the tidy verbs. `filter(...)` and other methods should just work without
the extra suffix.
* Perhaps you want to reorganize the class definition when doing `activate`?
This will allow you to use `filter_*` based on the first class in line.

Minor:
* Use `character()` as argument defaults to avoid checking for `missing` in
`BiocSet_from_elementset`
