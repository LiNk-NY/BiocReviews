## R

* The first two conditions in `checkInferenceEngine` are the same. Perhaps
you meant to add a different method (`centroidHomFold`)?
* Use double vertical bars `||` to ensure that conditions on either side
evaluate to length 1 (`R/ncRNAtools_checkInputFunction:L#32`).
* I would recommend that you keep warnings and error messages as brief as
possible and within the default R width (`getOption("width") == 80`).
* Use `globalVariables` function to avoid using `Var1 <- NULL` in every
plotting function.
* Avoid saving plots for the user. Remove `filename` argument. The user should
be able to call `ggsave` themselves with the return of your function.
* Use the alpha-numeric regex class to match numbers and letters `[[:alnum:]]`
* Good use of `GRanges`
* Avoid using `sapply`. Use either `lapply` or `vapply` if the return is
predictable.
* Format the code so that it fits the 80 character width limit where possible.
* (optional) Create helper functions for repetitive operations, for example
`function(x, text) { unlist(strsplit(x, text)) }`
* Avoid downloading data in the examples (see vignette section above)
* It would be good to have more unit tests in the package.

Minor:
* To avoid the 'cyclomatic' complexity, use strategies that save you space
(e.g., `match(method, c("centroidFold", "centroidHomFold", "IPknot"))` in
`R/ncRNAtools_otherSupportFunctions.R:L#4-12`)
* Avoid using all the letters in the alphabet by doing `A-Z` inside the regex
or using regex classes.
* Use 4 space indentation
