## R

* If possible, avoid acronyms `CCT` and opt for descriptive names.
* Reduce the cyclomatic complexity in functions such as `CCT`, `findBlocksT`,
if possible. See the `cyclocomp` package.
* Data looks good.
* Remove all `;` from the R code. It is not necessary when there adequate
space (e.g., `findEpitopeSegments`).
* Avoid the 'copy and append' method when using `for` loops in
`getClusterSegments` and other functions. We recommend the 'allocate and fill'
method. For example:
```r
result <- vector("numeric", length = 10)
for (i in seq_along(result))
    result[i] <- ...
```
* Avoid `1:n` sequence generation, use `seq_along` or `seq_len`
* Remove exclamation points in error messages, e.g., `!`.
* Avoid code repetition in `wilkMin/Max*Meta` functions. It seems that they
can be combined into a single function with `n` and `min/max` parameters.
