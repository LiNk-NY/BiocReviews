# visiumStitched #3501

* Overall the package is well put together. Increase robustness by using
named indices rather than numeric ones. Consider migrating the code to
other readers as this `SpatialExperiment` might change in the future.

## DESCRIPTION

* Looks good.
* Consider removing the `LazyData: true` line. Any data should be loaded with
`data(...)`.

## NAMESPACE

* Use more common names for functions, e.g., `spe_to_seurat` can `as.Seurat()`.
* Consider renaming `build_spe` and avoid abbreviations.
* Add `Seurat` to the `Suggests` field instead.

## vignettes

* Consider wrapping text to 80 characters for readability and ease of
maintenance.
* Remove `Sys.time()` calls from vignette. The BBS will provide this
information in the report.
* Remove installation instructions that use `remotes`. You should be using
`BiocManager::install()` instead.
* The interface is a bit awkward `build_spe`. It seems like `sample_info`
requires a lot of user intervention.
* Consider using other import tools as `read10xVisium` may be deprecated soon,
e.g., `VisiumIO`, `SpatialExperimentIO` and others. Disclaimer: I maintain
`VisiumIO`. Feel free to ask on the \#spatial Bioc slack channel.
* Remove extra code in the Reproducibility section of the vignette e.g.,
"extract R code", "wallclock" etc. Keep the session information.

## R

* Perhaps reduce the complexity of the exampels so that they are easier to
read and run.
* Avoid saving `Rds` files from the examples. Use cached data in conjunction
with `BiocFileCache`, if possible e.g., in `build_spe` and others.
* Avoid using index subsetting where possible and use names instead, e.g.,
`coords_xy[, 1]`.
* Avoid creating folders for the user and assume that they are already
available; error when not e.g., in `rescale_fiji_inputs`.
* Add a package check to the `spe_to_seurat` function e.g., with
`BiocBaseUtils::checkInstalled()`. Consider re-using previously written
`spe_to_seurat` functions.
