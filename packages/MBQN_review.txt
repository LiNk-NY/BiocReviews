## R

* Use a more specific name for contained data
* Use temporary files to save data in `get_pxdfile` and other functions
instead of the working directory
* Use `message` instead of `print`
* Use `rename` instead of `file.move` and remove dependency
* `loadFile` is a generic name and it should be more specific
* You should use an appropriate Bioconductor class such as
`SummarizedExperiment` to store the return value of `loadFile`
* Consider using `BiocFileCache` for downloading and storing PRIDE data
* Use the more robust `seq_along` instead of `seq(1, x)`
* Do not save figures without the user's explicit consent. Avoid the `filename`
argument and just plot to the graphics device. Not all users want a `pdf` of
figures.
* `ifelse` calls in the `mbqnBoxplot` options can be replaced with
`x <- if(is.null(x)) value`, you can also create a helper function to avoid
repetitive code
* 'Demo' and 'example' functions are not needed when you have robust examples.
You should provide these demos in the example pages of the documentation
* You don't need to use double `::` in your own package `MBQN::get_kminmax`
* Remove `save_fig` and `show_fig` arguments, plotting functions will always
plot there should be no option to not plot. You may have to separate analysis,
and plotting functions in `mbqnPlotAll`
* Delete `set.seed` in `mbqnSimuData` and `mbqnSimuDistortion`. It has no
place inside functions even if it is a comment.


Minor:
* Use `file.exists` instead of `length(list.files(...))`
* Make a variable out of the `file.path` call and reuse it
* Use `all(is.na(r))` to check for NA values across the row instead of the
series of functions in `R/loadFile.R#63`
* Avoid using `$` symbols that use fuzzy matching and extract with `[[`
* Use `nrow` and `ncol` instead of `dim(x)[1]` and  `dim(x)[2]`
* Make `FUN` in `mbqn` a functional argument rather than matching the function
internally as is done by convention.
