## R

* Avoid over-using switch and nesting switch calls by creating a lookup table
for `.checkNumArgs`. The only difference in `numeric` and `integer` switches
is the text "numeric" and "integer" in the error message. The look up table
can be a `list` (roughly):

```r
argBounds <- c(5, 7)
gt <- list(op = function(x) x <= argBounds[1], cls = class,
    desc = function(x) "greater than")
lapply(gt, function(f) f(3))
# $op
# [1] TRUE
#
# $cls
# [1] "numeric"
```
* Avoid nested for loops where possible (`R/external.R`)
* Do not use `writeLines` or `print` in package code. If you need to show
a message, use `message`.
* Your error messages in `.myGetBM` should indicate that `sitadela`
should be tagged when reporting at https://support.bioconductor.org.
* The functions you are writing look quite manual and laborious. Is there a
way to interface with bioMaRt using the `xml` package or another package?
* Using numeric indices to subset can be risky when the data change. We advise
using names to subset data e.g., `ann[, "score"]` rather than `ann[, 1]` (from
`R/fromtxdb.R`; and avoid repeating this code if possible)
* Minimize the amount of copy-and-paste code by creating a helper function
and reusing it. Hehe is one block of code that gets repeated quite often:

