# R

* Remove commented code in `module_04_diff_analysis.R`.
* Reduce code repetition where possible, e.g. in `R/module_03_data_explor.R`.
* Avoid using `initialize` methods in `RFLOMICS-Class.R` for reasons listed
here: https://stackoverflow.com/a/16248773/6558242
* Do not use `{}` in setGeneric due to performance issues. See :
https://adv-r.hadley.nz/s4.html#s4-generics
* Make sure that the method signature matches the generic signature e.g.,
include all arguments e.g., `...` in `resetRflomicsMAE` and others.
* Ensure that functions and methods do not write to disk by default without
the user's consent / prompt e.g., in  `generateReport`.
* Use accessor functions in methods e.g., `colData(object)` instead of
`object@colData` in the `getDesignMat,RflomicsMAE-method`. Use also
`metadata()`, etc.
* Reduce repetitive code by isolating common operations, e.g., :
```r
if (raw) {
    object@metadata[["PCAlist"]][["raw"]] <-
        PCA(t(pseudo), ncp = ncomp, graph = FALSE)
} else {
    object@metadata[["PCAlist"]][["norm"]] <-
        PCA(t(pseudo), ncp = ncomp, graph = FALSE)
}
```
becomes

```r
rawnorm <- if (raw) "raw" else "norm"
metadata(object)[["PCAlist"]][[rawnorm]] <-
    PCA(t(pseudo), ncp = ncomp, graph = FALSE)
```
in the `runOmicsPCA,RflomicsSE-method`.
* Run spell checking on the text, e.g., `replicates` instead of `replicats`
in the `checkExpDesignCompleteness,RflomicsSE-method`.
* Consider using a lookup table `data.frame` or `list` for default values of
`transformation`, `normFactors` and other values in
`runCoExpression,RflomicsSE-method` rather than encoding them in a series of
`ifelse` statements. This way it is easier to maintain and easier to review.
* Consider supporting existing `MultiAssayExperiment` / `SummarizedExperiment`
objects as inputs to your constructor function. The user should not need to
deconstruct and re-construct the object for your constructor function.
