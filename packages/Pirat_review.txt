easier to review and read.

* Add a description to all fields in the data documentation (`@format` field
in `R/doc-data.R`)
* Remove or fix commented code in the `estimate_gamma()`,

multiple or `|` statement. Provide a more accurate description in the

function.

the shiny functionality.

user.
* Avoid using `cat` in functions that are not `print` or `show` methods, e.g.,
in `pipeline_llkimpute`.

user can see the full list of options in the usage section of the
documentation.

`get_indexes_embedded_prots`

fragile if any of those components begin to fail.
