## R

* Consider using `colSums` and `rowSums` instead of `apply(x, 2 / 1, sum)`.
* `exprs` alludes to deprecated functionality. If this is not internal,
use a keyword that is more descriptive in `R/bnem_low.R`.
* Avoid writing files to the disk for the user (e.g., in `addEdge`). Write
to the temporary directory using `tempdir()`.
* If possible, avoid nested for loops (in `adj2dnf`).
* Although entertaining, perhaps it is better to remove error messages such as
"I'm sorry, Dave. I'm afraid I can't do that." in `checkMethod`.
* Avoid using the `@` accessor see the package guidelines:
https://bioconductor.org/developers/package-guidelines/#rcode
* Using `print` is not recommended in package code unless in a `show` method.
Use `message` instead.
* (optional) Consistently use the assignment operator `<-` and avoid `=` for
assignment.
* Workflows related to a publication are best published in a separate
repository or package (`processDataBCR`).
* It would be more consistent to have a constructor that uses the same name
as the class (`bnemBs` -> `bnembs`)
* Avoid copying and pasting code (e.g., use `glob2rx` in `gene2prot`).

Minor:
* `dim(x)[2]` corresponds to `ncol`
* Avoid redundance by removing `paste` from inside `stop`, `message`, etc.
* In some instances you don't need to convert logical to numeric index (e.g.,
`which(Epos < -beta)` can simply be `Epos < -beta` based on `Epos[wrongPos]`)
unless there is `NA`.
