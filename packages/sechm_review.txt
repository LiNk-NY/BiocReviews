## R

* Avoid using `eval(parse(...))` in `resetAllSechmOptions`. You can simply
list and set them to `NULL`.
* Would `.getDef("assayName")` return the value `assay` in the `switch`
function? These should probably agree to avoid errors.
* It may be better to create a package-wide environment / object for the
various options. See here for an example:
https://github.com/Bioconductor/SOUNDBoard/blob/master/R/zzz.R.
* Deprecations in un-released packages are not usually necessary. If you wish,
you can deprecate arguments with `.Deprecated`.
* Describe what type of data can `ses` contain.
* The code looks quite complicated and the number of arguments in `sechm`
can overwhelm any user. Try to reduce the number of specifiable arguments in
the function calls, if possible.
