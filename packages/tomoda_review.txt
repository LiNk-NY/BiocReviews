## R

* `CreateTomo` should support both `matrix` and `SummarizedExperiment` inputs.
* You can also have a normalize / scale argument in the `CreateTomo` function.
* Consider renaming `CreateTomo` to something like `checkTomo` as 'Tomo' is not
really a class or subclass of `SummarizedExperiment`.
* Add more details in the documentation to what operations are performed on
the data as part of the `CreateTomo` function.
* Given that `SummarizedExperiment` is the main data representation. The
`GetMatrix` function is repeated code for `assay(se, i = "counts")` and should
be removed. The documentation should inform users to use the `assay` function
to extract a matrix.
* Avoid using `cat` and use `message` instead to notify the user.
* `HClust` and `KMeans` seem to be thin wrappers for `hclust` and `kmeans` and
consider including the convenience functions inside the main `CreateTomo`,
if at all.
* Avoid hardcoding the assay name in `FindPeakGene` (`assays(object)$scaled`).
Allow the user to input the scaled assay name (`scaled`) using an argument.
This applies to other functions that use `assays(object)$scaled` etc.
* Use `assay` rather than `assays` when extracting one
* Use the allocate and fill strategy in `FindPeakGene` `for` loop when
`nperm > 0`.
