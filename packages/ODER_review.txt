## R

* Keep the code within the 80 column width limit.
* Remove `dontshow` section from `R/annotatER.R`. Examples run in sequence so
you can run that code in a previous/current example and use the created object.
* Avoid using `exists` and ensure that the object exists based on above
comment.
* Some datasets need more descriptive names, "*_example" is not very helpful.
It is good practice to include study origin in the name, if possible.
* Use `message` instead of `print`.
* Avoid `1:n` sequence generation use `seq_len`/ `seq_along`
* Address `NOTE`s in `BiocCheck::BiocCheck()`
* Avoid nested `for` loops if possible.
* Do not use hardcoded values in code, e.g., `mcols(gtf)[["type"]]`. Add an
argument that lets the user specify the relevant column in the gtf file.
* Consider using the native R pipe operator `|>`.
* `ODER:::` (with triple colons) in same package code is rarely needed see
`R/ODER.R`.
* `file.exists` does not make sense for a URL in `.file_cache`. The use of
`BiocFileCache` is a bit awkward. It seems to only work for URLs and errors
out for non-existent files.
