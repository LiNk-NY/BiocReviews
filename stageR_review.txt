# stageR #432

I agree with Michael @lawremi, I don't see why you wouldn't be able to
extend `SummarizedExperiment` for the `stageR` class. You would benefit
from the already established structure.

## DESCRIPTION

* Please indicate the appropriate packages in the
imports and suggests field. If you require packages
for the vignette examples but not for the code
in the package, put them under the suggests field.
If you are using external functions for the code in your package
use the imports field. If you are building on top of other
functions outside of your package, indicate them in the
depends field. 

## NAMESPACE

* It seems like your accessor functions don't need to be prepended
with 'get'. It is a matter of preference but my suggestion would be
to remove 'get' and leave just the camelCase nouns. 

## Formatting

* Please make sure you're formatting your code to not exceed 80 column
widths. This makes it harder to read and to maintain.

## R

* Please follow the packages guidelines as listed in Bioconductor.org
* Avoid using, `1:length(x)` sequence generation, the `=` for setting
variables, sapply for looping.
* Don't repeat yourself, create helper functions when you need to run
similar operations under different 'methods', e.g. `.stageWiseTest`
* Use accessor functions to extract data within your class,
avoid using `@`
* Avoid using numeric indices for accessing data vectors, use more robust
names for extracting columns, e.g., `df[, "var1"]` vs. `df[, 1]`
* Define methods for your class so that you don't need to check for
S4 class membership because it will only dispatch to that class
(`.getResults`)
* You don't need to create an off function for your show method, just
include the function as part of the `setMethod` call

## vignettes

* Please include your dataset generation script under `inst/extdata`,
no need to include it in R (`constructDTUDataset.R`)
* Load data appropriately using `system.file` in your example.
* Avoid downloading data in the vignette
* Don't install packages programatically in the vignette, just provide
the library calls. Users will donwload them on their own.
* Use `require` to check availability of packages rather than
`installed.packages()`
* Use meaningful variable names in the vignette, L, v, and d are not very
illustrative.
